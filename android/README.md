# Claude OS — Pixel launcher

A real Android **launcher** controlled by Claude. Press home → you land in
Claude. Every screen is generated by the model in real time through a
dynamic UI tool.

## Install on your Pixel — easy path (no Android Studio)

GitHub Actions builds the APK in the cloud every time the code changes
and publishes it as a rolling pre-release. To install:

1. On your Pixel 7 Pro, open Chrome and go to:
   `https://github.com/oren001/elad-and-oren-pipeline/releases/tag/rolling-debug`
2. Tap the `app-debug.apk` asset to download.
3. Open the downloaded APK. Android may ask to "allow installs from
   this source" — say yes. Tap Install.
4. `Settings → Apps → Default apps → Home app → Claude OS`.
5. Press Home. Paste your Anthropic API key. You're in.

If a new build is in flight, the rolling release updates within ~5
minutes of any push to the `claude/custom-os-pixel-*` branch.

## Install — power-user path (Android Studio)

1. Install Android Studio (Hedgehog or newer).
2. `File → Open` → pick the `android/` folder.
3. Plug in your Pixel with USB debugging on (`Settings → About phone →
   tap Build number 7×` → enable USB debugging in Developer Options).
4. Hit Run ▶. Studio installs and launches it.

Either path stores your Anthropic key in encrypted DataStore on the
device only — it never leaves except as the `x-api-key` header on calls
to Anthropic.

## One-time enable (asked from inside the launcher)

- **Accessibility**: `Settings → Accessibility → Claude OS — control surface → On`.
  Lets Claude read what's on screen and tap/swipe in other apps so it can
  drive Spotify, WhatsApp, Maps, etc. on your behalf.
- **Notification access**: `Settings → Notifications → Device & app
  notifications → Claude OS — notification listener → On`.
  Lets the assistant see incoming texts/calendar/etc. so it can interrupt
  you when something genuinely important happens.

You can disable either at any time from the same screens.

## What works on day one

- Claude renders every screen via `render_ui` (dynamic UI DSL).
- Push-to-talk voice in (Android `SpeechRecognizer`) and TTS out.
- Camera capture (CameraX) with front/back switch.
- Slide-to-confirm for risky actions (payments, sends, calls).
- Personal memory in Room (facts + lessons), fed to the system prompt
  every turn. The OS gets sharper at knowing you over time.
- Tools that hit Android intents directly: place call, send SMS, open
  WhatsApp, navigate (Maps), open any installed app, web browse.
- Notification listener pipes incoming events to Claude as `[event]`
  messages; the model decides whether to call `interrupt_user` to
  surface a notification of its own.

## What's stubbed (returns "bridge not yet connected")

- Smart home (will plug into Home Assistant / Matter)
- Calendar read/write (will plug into the device Calendar provider)
- Photo library search (returns placeholders for now)
- Payments

## Architecture

```
UI (Jetpack Compose)
  Shell.kt            — top bar, dynamic content, mic/camera/text dock
  Renderer.kt         — UISpec (JSON tree) → Compose tree
  SlideToConfirm.kt   — Bunq-style swipe gesture
  CameraScreen.kt     — CameraX viewfinder + capture
  ApiKeyGate.kt       — first-run

Agent
  api/ClaudeClient.kt — tool-use loop hitting v1/messages
  api/Tools.kt        — tool registry (server- vs client-handled, risk tiers)
  api/SystemPrompt.kt — persona + memory + lessons
  api/UiSpec.kt       — kotlinx.serialization data classes for the UI DSL

Device
  ShellViewModel.kt   — turn driver, history, photo round-trip, gates
  tools/ToolRunner.kt — Android intents for navigate/call/sms/open_app/etc.
  voice/Voice.kt      — SpeechRecognizer + TextToSpeech
  memory/Memory.kt    — Room DB for facts + lessons
  services/OsAccessibilityService.kt
  services/OsNotificationListenerService.kt
```
